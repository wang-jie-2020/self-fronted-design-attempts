Concept 1 · Subagent

定义：主 Agent 派出的专职子任务执行者

价值

  • 并行推进，缩短总时长
  • 单任务单维度，减少混乱
  • 主 Agent 专注于决策汇总

使用条件

  • 子任务可以独立完成
  • 结果可单独验证
  • 合并后可形成决策
06

Subagent Example

PR 评审：三路并行，主 Agent 汇总

A · Correctness

  • 边界条件
  • 业务规则一致性

B · Performance

  • 热点路径耗时
  • 重复计算与 I/O

C · Security

  • 权限边界
  • 敏感信息暴露风险

输出格式：风险清单 + 优先级 + 修复顺序。

07

Split checklist

什么时候该拆？什么时候不该拆？

适合拆分

  • 多维度并行检查
  • 任务复杂度高
  • 可并行且弱依赖

不适合拆分

  • 任务极小（几行修复）
  • 步骤强依赖链
  • 协调成本高于收益
08

Concept 2 · Hooks

定义：在关键时机自动触发的流程闸门

解决的问题

  • “知道该做但总会忘”
  • 人为执行质量不稳定
  • 低级错误晚暴露

收益

  • 质量前移
  • 团队行为一致
  • 自动化守门可复用
09

Hook pipeline

前中后三段 Hook：最小投入，最大收益

Before
预执行检查

环境、范围、权限与风险命令确认。

During
过程拦截

敏感模式命中即阻断并给修复提示。

After
自动收尾

测试、摘要、规范检查一次完成。

好 Hook 三原则： 1) 可自动化且可判定 2) 失败时有可执行修复建议 3) 不制造提示噪音
10

Concept 3 · MCP

定义：模型接入外部工具与实时数据的协议层

模型层

负责理解、推理、规划。

协议层

统一工具调用与数据交换语义。

能力层

代码索引、文档、工单、内部 API。

MCP 的价值：结论可追溯，不靠模型“记忆猜测”。

11

Without vs With MCP

同一个问题，信息质量差一个等级

无 MCP

  • 靠记忆与盲搜拼接
  • 路径容易漏节点
  • 结果难复核

有 MCP

  • 结构化上下文+源码定位
  • 调用路径可验证
  • 结论可审计
示例："这个函数如何到达渲染层？" → trace/context/explore 组合得到可验证路径。
12

Concept 4 · Skill

定义：把高频任务固化为可触发的流程命令

本质

  • 不是快捷键，是流程模板
  • 输入/输出/质检可复用
  • 经验沉淀成团队资产

示例

/article-writing /code-review /verify
13

Skill design checklist

一个成熟 Skill 至少有四个部件

触发语义

何时必须触发，避免歧义。

输入契约

缺参时提问补齐，不猜测。

输出模板

结构可预期，可直接交付。

质量闸门

事实检查、格式检查、异常回退。

14

Concept 5 · CLAUDE.md

定义：项目级长期指令，约束行为并统一输出

应该写什么

  • 沟通与决策规则
  • 风险边界（密钥/破坏命令）
  • 检索策略与默认范围
  • 项目风格与交付格式

解决什么

  • 跨会话一致性
  • 多人协作默认对齐
  • 减少重复口头说明
15

Contract sample

可执行的 CLAUDE.md 要短、硬、可验证

## 沟通 - 先给方案，再解释 - 不清楚先提问 ## 风险 - 禁止提交密钥 - 破坏命令先确认 ## 检索 - 默认定向检索 - 未给路径先确认范围

常见误区

  • 写愿景，不写可执行约束
  • 规则互相冲突无优先级
  • 长期不维护，快速过期

48-hour case

Adoption roadmap

从个人可用到团队可复制：三层推进

L1 · 个人试用

L3 · 平台化

  • 把 Hook 产品化
  • 建立质量指标
